在C++20引入协程基础语法（`co_await`、`co_yield`、`co_return`）后，C++23和C++26进一步完善了协程相关特性（如更灵活的暂停/恢复机制、标准库扩展等）。目前市面上支持C++协程（无栈协程）的运行时库主要基于C++20及后续标准扩展，核心提供`awaitable`对象、调度器、任务管理等功能，以下是主流库的梳理：


### 1. **Boost.Asio**
- **特点**：Boost生态中最成熟的异步IO库，最新版本已深度支持C++20协程，通过`asio::awaitable<T>`类型实现协程与异步操作（网络、文件、定时器等）的无缝结合。
- **优势**：跨平台（Windows、Linux、macOS等），支持TCP/UDP、串口、信号量等多种IO类型，`co_await`可直接等待异步操作完成，无需回调。
- **适配性**：已兼容C++20/23协程特性，对C++26的扩展（如协程内存优化）正在跟进，是工业级项目的首选。


### 2. **libunifex**
- **背景**：由Facebook主导开发，目标是成为C++标准“执行模型”的基础，旨在统一异步/并发编程范式。
- **特点**：基于C++20协程，提供丰富的“调度器”（scheduler）和“组合器”（combinator），支持协程的串联、并行、条件分支等复杂逻辑，强调类型安全和性能。
- **适配性**：紧密跟进C++标准演进（包括C++26的协程扩展），是研究现代C++协程最佳实践的重要参考。


### 3. **cppcoro**
- **背景**：由C++协程标准提案核心参与者Lewis Baker开发，是早期C++20协程的标杆实现。
- **特点**：提供基础协程组件，如`task<T>`（异步任务）、`generator<T>`（迭代生成器）、`sequence<T>`（序列生成）等，API简洁，适合学习和轻量场景。
- **现状**：虽不再积极维护，但代码质量高，理念影响了后续很多库（如libunifex），仍可用于兼容C++20/23的项目。


### 4. **Folly（Facebook Open Source Library）**
- **背景**：Facebook的开源工具库，包含高性能协程模块。
- **特点**：基于C++20协程，提供`folly::coro::Task<T>`，支持异步IO、线程池调度、协程取消等，与Folly的其他组件（如`Future`）无缝集成，性能优化极致（针对多核和高并发）。
- **适配性**：优先支持C++标准的最新特性，对C++26的协程改进（如更高效的状态机生成）已有实验性支持。


### 5. **Microsoft STL 扩展**
- **背景**：微软在其STL中提供了协程相关扩展，主要面向Windows平台。
- **特点**：包含`std::experimental::generator<T>`（生成器）、`winrt::resume_background`（后台线程调度）等，可直接`co_await` Win32 API的异步操作（如文件IO、网络请求）。
- **适配性**：深度整合Windows生态，对C++26的协程内存模型优化（如减少堆分配）有早期支持。


### 6. **Qt 6+**
- **背景**：Qt 6开始引入对C++20协程的支持，主要服务于Qt应用的异步逻辑。
- **特点**：通过`QFuture`与协程结合，可`co_await` Qt的异步操作（如网络请求、数据库查询），简化GUI线程与后台任务的交互。
- **优势**：适合Qt生态项目，避免回调嵌套，提升代码可读性。


### 7. **coro（轻量级库）**
- **特点**：开源轻量级协程库（GitHub：[lewissbaker/coro](https://github.com/lewissbaker/coro)），专注于基础任务调度和`awaitable`实现，代码量小，适合嵌入小型项目。
- **适配性**：支持C++20/23，对C++26的新特性（如`std::coroutine_traits`扩展）有初步兼容。


### 总结
这些库均基于C++无栈协程模型（依赖`co_await`等语法），核心差异在于应用场景：
- 工业级异步IO首选**Boost.Asio**；
- 通用并发/调度逻辑考虑**libunifex**或**Folly**；
- 轻量学习或小型项目可尝试**cppcoro**或**coro**；
- 平台特定场景（Windows/Qt）优先对应生态库。

随着C++26标准化推进，这些库会逐步适配新特性（如协程性能优化、标准`awaitable`类型），建议关注其官方仓库的更新日志。
